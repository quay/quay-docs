= Viewing detailed logs

[role="_abstract"]
To inspect individual registry events for a user, organization, or repository in {productname}, you can retrieve detailed logs through the API. You can filter results by performer and date range.

.Procedure

. Use the `GET /api/v1/user/logs` API endpoint to return a list of log entries for a user. For example:
+
[source,terminal]
----
$ curl -X GET   -H "Authorization: Bearer <bearer_token>"   -H "Accept: application/json"   "<quay-server.example.com>/api/v1/user/logs"
----
+
You can also pass in the `performer` and `starttime/endtime` queries to obtain logs for a specific user between a specific time period. For example:
+
[source,terminal]
----
$ curl -X GET   -H "Authorization: Bearer <bearer_token>"   -H "Accept: application/json"   "http://quay-server.example.com/api/v1/user/logs?performer=quayuser&starttime=01/01/2024&endtime=06/18/2024"
----
+

. Use the `GET /api/v1/organization/{orgname}/logs` endpoint to return logs for a specified organization:
+
[source,terminal]
----
$ curl -X GET \
  -H "Authorization: Bearer <bearer_token>" \
  -H "Accept: application/json" \
  "http://<quay-server.example.com>/api/v1/organization/{orgname}/logs"
----

. Use the `GET /api/v1/repository/{repository}/logs` endpoint to return logs for a specified repository:
+
[source,terminal]
----
$ curl -X GET \
  -H "Authorization: Bearer <bearer_token>" \
  -H "Accept: application/json" \
  "http://<quay-server.example.com>/api/v1/repository/{repository}/logs"
----

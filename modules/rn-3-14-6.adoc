[id="bug-fixes-314-6"]
== {productname} 3.14.6 bug fixes

* link:https://redhat.atlassian.net/browse/PROJQUAY-10269[PROJQUAY-10269]. Previously, {productname} ran on Python version 3.9 WIth this release, Python has been upgraded to version 3.12. 

* link:https://redhat.atlassian.net/browse/PROJQUAY-9832[PROJQUAY-9832]. Previously, the regular expression search dropdown was removed. This removal impacted user experience, as the regular expression search option became unavailable in search fields on Quay 3.16, Firefox, and Chrome. With this release, the regular expression search dropdown has been re-enabled. As a result, users have access to improved pattern matching.

* link:https://redhat.atlassian.net/browse/PROJQUAY-9525[PROJQUAY-9525]. In this update, users inadvertently employed an unsupported data type with the function, triggering a runtime error. Consequently, the customized registry title was hidden from the new UI's browser tab. With this release, customized registry titles now appear on browser tabs, thereby boosting UI personalization.

* link:https://redhat.atlassian.net/browse/PROJQUAY-9517[PROJQUAY-9517]. Previously, a newly created Quay Organization was not immediately displayed on the Quay UI due to a missing automatic page refresh. With this update, the UI now automatically refreshes the *Organizations* page upon creating a new Quay Organization. As a result, the newly created organization is now automatically visible on the *Organizations* page.

* link:https://redhat.atlassian.net/browse/PROJQUAY-9514[PROJQUAY-9514]. Previously, users were unable to update team descriptions within the new Quay UI. With this release, the team description update within the new UI of Quay is now functional. As a result, users can now edit the team description by clicking on the team name in Quay.

* link:https://redhat.atlassian.net/browse/PROJQUAY-9468[PROJQUAY-9468]. Previously, an issue caused the *Total Registry Size* to incorrectly show as *0.00 KiB* in Quay. This inaccuracy affected the size calculation for users. With this release, Quay now correctly calculates the *Total Registry Size*, improving the precision of size tracking for users.

* link:https://redhat.atlassian.net/browse/PROJQUAY-9461[PROJQUAY-9461]. This update resolves an issue where un-validated user input allowed arbitrary code execution during the login process, resulting in the customized title being hidden on the Quay login page. We've now corrected the wrong data validation in user input to accept only valid formats. As a result, the customized UI login title improves user recognition.

* link:https://redhat.atlassian.net/browse/PROJQUAY-9423[PPROJQUAY-9423]. In this update, the `quotaregistrysizeworker` in LDAP-driven Quay deployments now correctly meets superusers from LDAP. Previously, end users experienced issues with registry calculation when using LDAP superusers with LDAP-driven Quay registries. This release resolves the issue, allowing end users to perform registry calculation with LDAP superusers.

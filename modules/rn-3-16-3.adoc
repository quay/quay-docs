[id="bug-fixes-316-3"]
== {productname} 3.16.3 bug fixes

* link:https://issues.redhat.com/browse/PROJQUAY-10609[*PROJQUAY-10609*]. Previously, image pulls failed with `cryptography.exceptions.UnsupportedAlgorithm: sha1 is not supported by this backend for RSA signing` because the RHEL 9 image deprecated RSA-SHA1 signing and the system cryptography policy blocked SHA-1 for RSA.
+
With this release, the cryptography policy is adjusted so that SHA-1 is supported for RSA signing where required, restoring successful image pulls.

* link:https://issues.redhat.com/browse/PROJQUAY-10557[*PROJQUAY-10557*]. Previously, the new UI could show a success notification when an LDAP restricted user failed to create an organization, which misrepresented the outcome of the operation.
+
With this release, notifications reflect whether organization creation succeeded or failed for LDAP restricted users.

* link:https://issues.redhat.com/browse/PROJQUAY-10384[*PROJQUAY-10384*]. Previously, after upgrading to {productname} 3.16.1, deployments using a Ceph RADOS Gateway (RGW) storage backend logged repeated `[storage.cloud]` OpenTelemetry errors, which cluttered logs and obscured useful diagnostics.
+
This release corrects the behavior so those repeated OpenTelemetry storage errors no longer fill the logs for Ceph RGW backends.

* link:https://issues.redhat.com/browse/PROJQUAY-9948[*PROJQUAY-9948*]. Previously, the new UI could display a success notification when a user tried to create an organization with a name that already existed, even though the request did not succeed.
+
With this release, the new UI shows an accurate notification when organization creation fails because the name is already in use.

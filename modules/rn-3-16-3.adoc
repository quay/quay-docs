== {productname} 3.16.3 bug fixes

* link:https://issues.redhat.com/browse/PROJQUAY-10609[*PROJQUAY-10609*]. Previously, image pulls failed due to a deprecated RSA-SHA1 algorithm in the RHEL-9 image. WIth this release, the RHEL-9 cryptography policy has been updated, enabling SHA-1 for SA signing. As a a result, the SHA-1 algorithm is now supported, and images pull without issue. 

* link:https://issues.redhat.com/browse/PROJQUAY-10557[*PROJQUAY-10557*]. Previously, LDAP user filter triggered a success notification for a failed organization creation, which could cause confusion. This issue has been resolved by using updating .mutate to .mutateAsync() in the createOrganization API function. As a requlest, restricted LDAP users are now correctly informed that organization creation failed due to certain restrictions. 

* link:https://issues.redhat.com/browse/PROJQUAY-10384[*PROJQUAY-691038434*]. Previously, registry logs showed repeated error-level log entries originating from the storage cloud layer when OTEL instrumentation was enabled. Although no functional impact was observed, this error still produced faulty log information. This issue has been resolved by downgrading OTEL logs from ERROR to DEBUG. As a result, repeated error-log entries from OTEL instrumentation no longer crowd log entries. 

* link:https://issues.redhat.com/browse/PROJQUAY-9948[*PROJQUAY-9948*]. Previously, users attempting to create a Quay Organization with an existing name received a misleading success notification, causing failed backend creations and potential data inconsistency. With this release, the new Quay UI now immediately shows an error message when an organization name already exists, preventing creation of duplicate organizations and ensuring immediate error notifications.

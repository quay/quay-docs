= Retrieving the default configuration

[role="_abstract"]
To retrieve the default {productname} configuration when you do not have an existing config bundle, you can start the container in config mode and call the configuration API.

.Prerequisites

* You have installed either `podman` or `docker`.
* You can reach the host and port where you run the configuration tool.

.Procedure

. Start the {productname} container in configuration mode. For example:
+
[subs="verbatim,attributes"]
----
$ sudo podman run --rm -it --name quay_config \
  -p 8080:8080 \
  {productrepo}/{quayimage}:{productminv} config secret
----

. Retrieve the default configuration by using the `config` endpoint of the configuration API:
+
[source,terminal]
----
$ curl -X GET -u quayconfig:secret http://quay-server:8080/api/v1/config | jq
----
+
Example output
+
[source,json]
----
{
  "config.yaml": {
    "AUTHENTICATION_TYPE": "Database",

= Understanding usage logs

[role="_abstract"]
By default, {productname} stores usage logs in its database and shows them in the web UI. You can query those logs in PostgreSQL and map action types by kind_id.

Usage logs appear at the organization and repository levels, and in the *Superuser Admin Panel*. Database logs capture a wide range of events in {productname}, such as account plan changes, user actions, and general operations. Log entries include information such as the action performed (`kind_id`), the user who performed the action (`account_id` or `performer_id`), the timestamp (`datetime`), and other relevant data associated with the action (`metadata_json`).

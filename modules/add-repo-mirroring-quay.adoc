= Adding repository mirroring to {productname}

[role="_abstract"]
To synchronize external registry content into {productname}, you can enable repository mirroring in `config.yaml` and start the mirroring worker on a registry node.

Enabling repository mirroring allows you to create container image repositories
on your {productname} cluster that match the content of a selected
external registry, then sync those repositories on a regular schedule and on demand.

.Procedure

. Set `FEATURE_REPO_MIRROR: true` in your `config.yaml` file. For optional mirroring fields, see _Additional resources_.

. Start the repository mirroring worker on a {productname} node:
+
[subs="verbatim,attributes"]
----
$ sudo podman run -d --name mirroring-worker \
  -v /mnt/quay/config:/conf/stack:Z \
  -v /root/ca.crt:/etc/pki/ca-trust/source/anchors/ca.crt \
  {productrepo}/{quayimage}:{productminv} repomirror
----
+
If you do not use a custom CA certificate, omit the `-v /root/ca.crt:...` line.

. Restart the {productname} registry containers so they load the updated configuration.

. Create mirrored repositories in the {productname} web UI as described in _Additional resources_.

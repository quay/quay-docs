== {productname} 3.16.2 bug fixes

* link:https://issues.redhat.com/browse/PROJQUAY-10155[*PROJQUAY-10155*]. Previously, the {productname} on-site installation lacked the `/health` endpoint in version 3.16.1, resulting in a `404` error and impairing health check functionality. With this update, the `/health` endpoint returns a valid response, enhancing health check functionality. 

* link:https://issues.redhat.com/browse/PROJQUAY-10120[*PROJQUAY-10120*]. Previously, the default{productname} Operator dashboard did not display metrics. With this update, the issue has been resolved and the default Quay dashboard in the {ocp} console correctly displays metrics. 

* link:https://issues.redhat.com/browse/PROJQUAY-[*PROJQUAY-9696*]. Previously, LDAP superusers were incorrectly restricted from creating organizations due to a fallback to `ConfigUserManager`. With this update, the LDAP result now holds authority for restricted users, enabling LDAP superusers to create organizations when other restricted users are blocked.

= Immutability policy overview

[role="_abstract"]
An immutability policy in {product-title} defines which image tags cannot be overwritten, modified, or deleted. You can apply policies at the tag, repository, or organization level, or use a regex pattern so that matching tags become immutable on push. Immutability policies can be created via the {productname} v2 UI or the API.

Immutability can be applied at the individual tag, repository, or organization level. {productname} administrators and users can define regex patterns to automatically mark matching tags as immutable upon push. Developers can trigger immutability directly from their build process by including the `quay.immutable=true` label in their Dockerfile/Containerfile.

When active, the system blocks tag overwrites (retargeting), manual or programmatic deletions, and auto-pruning by background workers. Changes to manifest labels associated with the immutable tag are also blocked.

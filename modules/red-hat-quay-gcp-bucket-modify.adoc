= Modifying your Google Cloud Platform object bucket

[role="_abstract"]
To enable virtual builds with Google Cloud Platform storage in {product-title}, you can configure cross-origin resource sharing (CORS) settings in your GCP bucket. This allows build workers to upload Dockerfiles and access build artifacts.

[NOTE]
====
Currently, modifying your Google Cloud Platform object bucket is not supported on IBM Power and IBM Z.
====

.Procedure 

. Use the following reference to create a JSON file for your specific CORS needs. For example:
+
[source,terminal]
----
$ cat gcp_cors.json
----
+
.Example output
[source,json]
----
[
    {
      "origin": ["*"],
      "method": ["GET"],
      "responseHeader": ["Authorization"],
      "maxAgeSeconds": 3600
    },
    {
      "origin": ["*"],
      "method": ["PUT"],
      "responseHeader": [              
              "Content-Type",
              "x-goog-acl",
              "origin"],
      "maxAgeSeconds": 3600
    }
]
----

. Enter the following command to update your GCP storage bucket:
+
[source,terminal]
----
$ gcloud storage buckets update gs://<bucket_name> --cors-file=./gcp_cors.json
----
+
.Example output
[source,terminal]
----
Updating                                                                                                  
  Completed 1    
----

. You can display the updated CORS configuration of your GCP bucket by running the following command:
+
[source,terminal]
----
$ gcloud storage buckets describe gs://<bucket_name>  --format="default(cors)"
----
+
.Example output
[source,yaml]
----
cors:
- maxAgeSeconds: 3600
  method:
  - GET
  origin:
  - '*'
  responseHeader:
  - Authorization
- maxAgeSeconds: 3600
  method:
  - PUT
  origin:
  - '*'
  responseHeader:
  - Content-Type
  - x-goog-acl
  - origin
----

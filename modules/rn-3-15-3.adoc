= RHSA-2026:15211 - {productname} 3.15.3 release

Issued 2026-09-18

{productname} release 3.15.3 is now available with Clair {clairproductminv}. The bug fixes that are included in the update are listed in the link:https://access.redhat.com/errata/RHSA-2026:1942[RHSA-2026:1942] advisory.

[id="bug-fixes-315-3"]
== {productname} 3.15.3 bug fixes

* link:https://issues.redhat.com/browse/PROJQUAY-9646[PROJQUAY-9646]. In this update, a repository path containing more than two segments previously triggered a `ValueError` in `util/jinjautil.py`. Consequently, users did not receive email notifications for repositories with multiple slashes in their names. With this release, the split function in `util/jinjautil.py` now accommodates repositories with multiple slashes, ensuring that email notifications for such repositories function correctly.

* link:https://issues.redhat.com/browse/PROJQUAY-9525[PROJQUAY-9525]. In this update, users might have inappropriately utilized the function with non-standard input, leading to unforeseen behavior. Consequently, the customized registry title was not apparent on the Quay browser tab in the new UI. 
+
With this release, database connection settings have been addressed to mitigate intermittent connection issues. As a result, the customized registry title is now visible in the browser tab, improving UI clarity.

* link:https://issues.redhat.com/browse/PROJQUAY-9677[PROJQUAY-9677]. Previously, user data on the new organization page was previously lost after prolonged inactivity due to the UI session expiring. This issue of data loss on the new organization page has been resolved ensuring that data is no longer lost after extended periods of inactivity.

* link:https://issues.redhat.com/browse/PROJQUAY-9522[PROJQUAY-9522]. Previously, the new UI no concealed customized registry titles. With this release, the customized registry title is visible on the Quay new UI login page. 

* link:https://issues.redhat.com/browse/PROJQUAY-9513[PROJQUAY-9513]. In Quay 3.10.1 and earlier versions, creating a new Quay Organization on the new UI did not automatically display the newly created organization, making it invisible to the user. This issue has been resolved in Quay 3.15.3, and the new UI now automatically refreshes to display new Quay Organizations, ensuring they are visible after creation.

* link:https://issues.redhat.com/browse/PROJQUAY-9512[PROJQUAY-9512]. Previously, a new UI actively prevented users from updating team descriptions by clicking on team names. This issue resulted in an inability for users to update team descriptions in Quay 3.15.1. With this release, the issue in the new UI has been actively addressed, enabling users to click on team names for description updates. As a result, users can now modify team descriptions following creation.

* link:https://issues.redhat.com/browse/PROJQUAY-9460[PROJQUAY-9460]. In this release, thehe `QuotaSizeWorker` condition has been resolved to rectify the incorrect conditions. This fix has restored the `Total Registry Size` calculation, ensuring the data displayed for users in Quay 3.15.2 is now exact.

* link:https://issues.redhat.com/browse/PROJQUAY-9299[PROJQUAY-9299]. With this release, using an outdated Clair version resulted in disrupted user experience. This release resolves the issue by actively updating the Clair version.


* link:https://issues.redhat.com/browse/PROJQUAY-9272[PROJQUAY-9272]. With this release, the regular expression search has been added as an option in search fields. As a result, the advanced search functionality via regular expressions is now available.

* link:https://issues.redhat.com/browse/PROJQUAY-9012[PROJQUAY-9012]. Previously, users could not save the remote registry configuration due to unauthorized access to g`hcr.io`, resulting in `401` and `403` errors when saving the proxy configuration. This issue has been resolved by enabling the saving of proxy config for `ghcr.io` with `401` and `403` responses. This enhancement allows users to successfully save the remote registry configuration for `ghcr.io`, thereby improving the setup experience.

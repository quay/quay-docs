= Migrating from Entra ID v1.0 to v2.0

[role="_abstract"]
To migrate from Microsoft Entra ID v1.0 to v2.0 without interrupting clients in {productname}, you can set the v2.0 discovery endpoint and temporarily list both issuers in `OIDC_ISSUERS`.

.Procedure

. Set `OIDC_SERVER` to the v2.0 endpoint (`https://login.microsoftonline.com/<tenant-id>/v2.0/`).

. Add both issuer URLs to `OIDC_ISSUERS`:
+
[source,yaml]
----
  OIDC_ISSUERS:
    - https://sts.windows.net/<tenant-id>/
    - https://login.microsoftonline.com/<tenant-id>/v2.0
----

. Update upstream clients to v2.0 tokens.

. After all clients use v2.0, remove the v1.0 issuer from `OIDC_ISSUERS`.
